# 三层混合架构实施蓝图与技术攻坚方案

版本：v0.1（评审草案）  
范围：Codex-Unity MCP（L2 sidecar + L3 Unity Editor）

## 0. 目标与设计原则

### 0.1 核心目标
- 放弃“逐个 Unity API 注册 MCP Action”的线性扩张路线。
- 建立三层混合能力栈：
  - L1 原语层（高频确定性动作）
  - L2 泛化层（`set_serialized_property` + `get_serialized_property_tree`）
  - L3 专科层（子系统语义化能力）
- 在泛化层上线前，先完成两条底层防线：
  - 外部协议对 LLM 隐藏 `legacy_stringified_action_data`。
  - 原子回滚测试基座 `AtomicActionTestBase`。

### 0.2 约束与现实基线（基于当前代码）
- 当前外部写入协议已禁止 `legacy_stringified_action_data`，要求 `action_data` object（`E_ACTION_DATA_STRINGIFIED_NOT_ALLOWED`）。
- L2->L3 线缆仍通过 `legacy_stringified_action_data` 内部桥接（`turnPayloadBuilders -> runtimeUtils -> Unity VisualLayerActionItem`）。
- L3 执行器当前以 `McpVisualActionContext.TryDeserializeActionData` 反序列化 `legacy_stringified_action_data`。
- 原子事务基建已存在：`UndoGuard + CompositeTransactionRunner + RollbackVerifier`。

## Phase 0：基础设施护航（必须先做）

### 0-A 协议隔离：LLM 永远看不到 `legacy_stringified_action_data`
- 目标：`legacy_stringified_action_data` 完全退化为内部实现细节，不出现在对 LLM 的 schema/示例/错误建议中。
- 执行项：
  1. 清点并封堵外泄面：
     - tool schema 示例文本
     -错误消息模板
     -日志回显（尤其 parse fail）
     -状态回包中的 action payload
  2. 统一对外写协议：`action_data` object only。
  3. 在 sidecar 增加 `wire_contract_version`（建议 `r16.wire.v2`）并在 `/mcp/capabilities` 中宣告。

### 0-B L2->L3 线缆升级（去双重转义）
- 目标：消除“JSON 字符串嵌套 JSON”的双重转义链路。
- 建议落地策略（兼容迁移）：
  1. 新增线缆字段：`legacy_marshaled_action_data`（base64url + utf8 canonical json）。
  2. L3 优先解析 `legacy_marshaled_action_data`，缺失时回退 `legacy_stringified_action_data`。
  3. L2 在 Unity capability 确认支持后停止发送 `legacy_stringified_action_data`。
  4. 稳定后删除回退路径。
- 说明：此策略不要求一次性重写所有 handler DTO，可在现有 `JsonUtility.FromJson<T>()` 框架下平滑迁移。

### 0-C 原子测试基座：`AtomicActionTestBase`
- 目标：所有标记 `undo_safety=atomic_safe` 的 action 必须通过统一原子回滚测试。
- 强制门禁：
  - 成功路径：提交并保留变更。
  - 失败路径：回滚无泄漏。
  - 漏注册 Undo：测试必须 fail closed（`E_COMPOSITE_ROLLBACK_INCOMPLETE`）。

### Phase 0 完成判定（DoD）
- 任何 MCP 外部输入/输出均不出现 `legacy_stringified_action_data`。
- `AtomicActionTestBase` 在 CI 中执行，并覆盖现有 atomic_safe 动作。
- 线缆兼容状态可观测：v1/v2 命中率、fallback 次数。

## 2. 核心攻坚：`set_serialized_property` C# 设计方案

## 2.1 工具定位
- 类型：MCP 写工具（Action）。
- 语义：对单个目标组件执行一组 `SerializedProperty` patch。
- 事务：默认 atomic，支持 dry-run。

## 2.2 建议 DTO（L2 外部协议）

```json
{
  "based_on_read_token": "...",
  "write_anchor": { "object_id": "...", "path": "Scene/..." },
  "actions": [
    {
      "type": "set_serialized_property",
      "target_anchor": { "object_id": "...", "path": "Scene/Canvas/Button" },
      "action_data": {
        "component_selector": {
          "component_assembly_qualified_name": "UnityEngine.UI.Image, UnityEngine.UI",
          "component_index": 0
        },
        "patches": [
          {
            "property_path": "m_Color",
            "op": "set",
            "value": {
              "kind": "color",
              "color": { "r": 1, "g": 0.2, "b": 0.2, "a": 1 }
            }
          }
        ],
        "atomic": true,
        "dry_run": false,
        "prefab_override_policy": "record"
      }
    }
  ]
}
```

## 2.3 建议 DTO（L3 线缆对象，供 C# 反序列化）

```csharp
[Serializable]
public sealed class SetSerializedPropertyActionData
{
    public SerializedComponentSelector component_selector;
    public SerializedPropertyPatch[] patches;
    public bool atomic;
    public bool dry_run;
    public string prefab_override_policy; // record | reject_on_prefab_instance | allow
}

[Serializable]
public sealed class SerializedComponentSelector
{
    public string component_assembly_qualified_name;
    public int component_index; // same-type index on target, default 0
}

[Serializable]
public sealed class SerializedPropertyPatch
{
    public string property_path;
    public string op; // set | insert | remove | clear
    public int array_index;
    public SerializedValue value;
}

[Serializable]
public sealed class SerializedValue
{
    public string kind; // int|float|bool|string|enum|vector2|vector3|color|object_ref|array

    public long int_value;
    public double float_value;
    public bool bool_value;
    public string string_value;

    public string enum_name;
    public int enum_index;

    public Vector2Dto vector2;
    public Vector3Dto vector3;
    public ColorDto color;

    public SerializedObjectReference object_ref;
    public SerializedValue[] array_items;
}
```

## 2.4 JSON -> SerializedProperty 映射策略

### 映射规则（首批支持）

| SerializedPropertyType | 输入 `kind` | 赋值策略 |
|---|---|---|
| Integer | `int` | `prop.intValue = checked((int)int_value)` |
| Float | `float` | `prop.floatValue = (float)float_value` |
| Boolean | `bool` | `prop.boolValue = bool_value` |
| String | `string` | `prop.stringValue = string_value ?? ""` |
| Enum | `enum` | 优先 `enum_name` -> `enumValueIndex`，回退 `enum_index` |
| Vector2 | `vector2` | `prop.vector2Value = new Vector2(...)` |
| Vector3 | `vector3` | `prop.vector3Value = new Vector3(...)` |
| Color | `color` | `prop.colorValue = new Color(...)` |
| ObjectReference | `object_ref` | 走对象引用解析器 |
| Array | `array` | `set/insert/remove/clear` 四种 op |

### 关键校验
- `FindProperty(property_path)` 为空：`E_PROPERTY_NOT_FOUND`。
- `prop.editable == false`：`E_PROPERTY_READONLY`。
- `kind` 与 `propertyType` 不兼容：`E_PROPERTY_TYPE_MISMATCH`。
- 超范围转换：`E_PROPERTY_VALUE_OUT_OF_RANGE`。

### 伪代码（核心写入流程）

```csharp
ExecuteSetSerializedProperty(context, dto):
  target = ResolveTargetByAnchor(context.RawAction.target_anchor)
  component = ResolveComponent(target, dto.component_selector)

  return AtomicRunner.Run("set_serialized_property", () => {
    Undo.RecordObject(component, "Codex set_serialized_property")
    so = new SerializedObject(component)

    foreach patch in dto.patches:
      prop = so.FindProperty(patch.property_path)
      ValidateWritable(prop)
      ApplyPatchByPropertyType(prop, patch)

    if (!dto.dry_run):
      so.ApplyModifiedProperties()
      PrefabUtility.RecordPrefabInstancePropertyModifications(component)
      EditorUtility.SetDirty(component)
      EditorSceneManager.MarkSceneDirty(component.gameObject.scene)

    return Success
  })
```

## 2.5 对象引用难题（ObjectReference）

### Payload 设计

```json
"object_ref": {
  "ref_kind": "scene_anchor",  
  "scene_anchor": { "object_id": "...", "path": "Scene/Root/Target" },
  "asset_guid": "",
  "asset_path": "",
  "sub_asset_name": "",
  "expected_type": "UnityEngine.Texture, UnityEngine.CoreModule"
}
```

### 解析策略
- `ref_kind = null`：赋 null。
- `scene_anchor`：通过现有 anchor 解析器找到场景对象；若属性目标类型是 `Component`，再按类型取组件。
- `asset_guid/asset_path`：`AssetDatabase.GUIDToAssetPath` + `LoadAssetAtPath(type)`。
- `sub_asset_name`：在 `LoadAllAssetRepresentationsAtPath` 中二次定位。
- 类型安全：`fieldType.IsAssignableFrom(resolved.GetType())`，否则 `E_OBJECT_REF_TYPE_MISMATCH`。

### 关键错误码
- `E_OBJECT_REF_NOT_FOUND`
- `E_OBJECT_REF_TYPE_MISMATCH`
- `E_OBJECT_REF_KIND_INVALID`

## 2.6 原子性与 Undo 兼容

### 设计原则
- `set_serialized_property` 默认 `atomic=true`。
- 与现有 `UndoGuard` / `CompositeTransactionRunner` 对齐，不新增并行事务体系。

### 实施建议
1. 抽象 `AtomicActionExecutor.RunAtomic(...)`：
   - 内部复用 `UndoGuard.BeginGroup/Commit/Rollback`
   - 复用 `RollbackVerifier` 做 fail-closed 检查
2. `set_serialized_property` 作为 atomic-safe action 注册。
3. 任何 patch 中途失败 -> 整 action 回滚。

### 与复合动作融合
- 当 `composite_visual_action` 步骤包含 `set_serialized_property` 时：
  - 不再开启嵌套 Undo group（避免双层回滚污染）。
  - 由外层事务统一提交/回滚。

## 3. Schema 防爆：`get_serialized_property_tree` 懒加载设计

## 3.1 设计目标
- 只返回“当前所需节点层级”，不返回全量字段树。
- 支持预算截断（node/char）与 cursor 分页。
- 返回可写性与值摘要，而非完整大值。

## 3.2 Query 请求 DTO

```json
{
  "query_type": "get_serialized_property_tree",
  "payload": {
    "target_anchor": { "object_id": "...", "path": "Scene/..." },
    "component_selector": {
      "component_assembly_qualified_name": "UnityEngine.UI.Image, UnityEngine.UI",
      "component_index": 0
    },
    "root_property_path": "",            
    "depth": 1,
    "after_property_path": "",           
    "page_size": 64,
    "node_budget": 128,
    "char_budget": 12000,
    "include_value_summary": true,
    "include_non_visible": false
  }
}
```

## 3.3 Query 返回 DTO（建议）

```json
{
  "ok": true,
  "read_token": { "token": "..." },
  "data": {
    "component": {
      "type": "UnityEngine.UI.Image, UnityEngine.UI",
      "target_path": "Scene/Canvas/Button"
    },
    "root_property_path": "",
    "depth": 1,
    "returned_count": 42,
    "truncated": true,
    "truncated_reason": "CHAR_BUDGET_EXCEEDED",
    "next_cursor": "m_Material",
    "nodes": [
      {
        "property_path": "m_Color",
        "display_name": "Color",
        "property_type": "Color",
        "is_array": false,
        "array_size": 0,
        "depth": 0,
        "writable": true,
        "read_only_reason": "",
        "value_summary": "(1.00,0.20,0.20,1.00)",
        "has_visible_children": false
      }
    ]
  }
}
```

## 3.4 懒加载执行策略

1. 根定位：`root_property_path == ""` 表示组件根；否则 `FindProperty(root_property_path)`。
2. 深度受控遍历：只遍历到 `depth`。
3. 预算中断：到达 `node_budget` 或 `char_budget` 即截断并给 `next_cursor`。
4. 值摘要策略：
   - 标量：直接格式化。
   - 长字符串：截断。
   - 数组：仅长度 + 前 N 项摘要。
   - ObjectReference：输出 `type + name + path/guid` 摘要。

### 伪代码

```csharp
GetSerializedPropertyTree(req):
  component = ResolveComponent(...)
  so = new SerializedObject(component)
  root = string.IsNullOrEmpty(req.root_property_path) ? so.GetIterator() : so.FindProperty(req.root_property_path)

  iter = root.Copy()
  end = root.GetEndProperty()
  while iter.NextVisible(true) && !SerializedProperty.EqualContents(iter, end):
    if !WithinDepth(iter, root, req.depth):
      continue
    node = BuildNodeSummary(iter)
    if ExceedBudget(node):
      return TruncatedWithCursor(iter.propertyPath)
    append(node)

  return Success(nodes)
```

## 4. 潜在踩坑预警（重点）

## 4.1 Prefab / Variant 覆盖链
- 在 Prefab 实例上改字段会形成 override；在 prefab asset 上改则影响全局。
- 对 variant 链路要避免误写 base prefab。
- 必须在结果中明确：
  - 写入对象是 Scene Instance 还是 Prefab Asset
  - 是否新增 override

## 4.2 不可写或危险字段
- `m_Script`、部分内部引擎字段不可写。
- 某些字段可写但写后立即被系统回填（Layout/Animator/NavMesh 等）。
- 必须 fail fast：`E_PROPERTY_READONLY` 或 `E_PROPERTY_RUNTIME_OWNED`。

## 4.3 ManagedReference 与多态序列化
- `SerializedPropertyType.ManagedReference` 复杂度高，MVP 建议先只读不写。
- 否则极易出现类型元数据不一致与反序列化崩溃。

## 4.4 Array / List 边界
- 插入删除会导致索引漂移，组合 patch 顺序必须稳定。
- 对 `Array.data[i]` 操作应采用“先高索引后低索引”的删除策略。

## 4.5 ObjectReference 跨域问题
- Scene 对象引用到 Asset 字段、或反向引用，类型虽然是 `Object` 也可能在运行时失效。
- 子资源（Sprite/AnimationClip）解析失败率高，需明确 `sub_asset_name` 规则。

## 4.6 Undo 与 OnValidate 副作用
- `ApplyModifiedProperties` 可能触发 `OnValidate`，产生连锁改动。
- 原子回滚验证应关注“脏场景集合”和“对象泄漏”两类副作用。

## 4.7 多对象编辑与 Domain Reload
- 当前 action 语义是单目标对象，禁止“隐式批量”。
- 编译重载窗口期写入必须拒绝（复用现有 `unity not ready` 门禁）。

- 把“可观测性字段”作为发布前置条件（失败原因分布、回滚触发率、prefab override 命中率）。
